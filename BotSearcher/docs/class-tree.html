Class Hierarchy

  • object: The most base type
    • GestorDeEjecucion.GestorDeEjecucion: Gestor para controlar la ejecucion de las sondas a traves de una cola de ejecucion.
    • GestorDeModulos.GestorDeModulos: Gestor encargado de gestionar todos los modulos antes de ser instanciados.
    • GestorDeResultados.GestorDeResultados: Gestor que permite gestionar los diferentes modulos escritores.
    • GestorDeSondas.GestorDeSondas: Gestor que permite la edicion de sondas.
    • InfoResultados.InfoResultados: Convencion de comunicacion de resultados entre la sonda y los modulos de escritura.
    • Instruccion.Instruccion: Clase base para todas las instrucciones ejecutables, excepto Exit.
      • Interprete.Interprete.Ayuda: La Ayuda es una Instruccion basica del interprete.
      • RepertorioGdM.CargarModulo: La instruccion CargarModulo permite cargar un moduos y anadirlos al gestor de modulos dada una etiqueta, una direccion de fichero o bien una direccion y un nombre de clase.
      • RepertorioGdE.Ejecutar: Inicia la ejecucion de las sondas.
      • RepertorioGdE.EliminarDeEjecucion: Elimina una sonda de nombre dado a la cola de ejecucion
      • RepertorioGdR.EliminarEscritor: Elimina una escritor de nombre dado.
      • RepertorioGdM.EliminarModulo: Elimina un modulo de nombre dado.
      • RepertorioGdS.EliminarSonda: Elimina una sonda de nombre dado.
      • RepertorioGdE.ListaEjecucion: Muestra por pantalla la lista de Sondas de la cola de ejecucion.
      • RepertorioGdR.ListarEscritores: Genera y muestra una Lista de Escritores.
      • RepertorioGdM.ListarModulos: Genera y muestra una Lista de Modulos.
      • RepertorioGdS.ListarSondas: Genera y muestra una Lista de Sondas.
      • RepertorioGdS.NuevaSonda: La instruccion NuevaSonda permite crear una sonda con un dispensador y un ejecutor previamente cargado.
      • RepertorioGdR.NuevoEscritor: La instruccion NuevoEscritor permite crear una sonda con un dispensador y un ejecutor previamente cargado.
      • RepertorioGdS.Parametros: Permite trabajar con los parametros de los dispensadores y de los ejecutores.
      • RepertorioGdR.ParametrosEscritor: Permite trabajar con los parametros de los dispensadores y de los ejecutores.
      • RepertorioGdE.Parar: Detiene la ejecucion de las sondas.
      • RepertorioGdE.PonerEnCola: Anade una sonda de nombre dado a la cola de ejecucion
    • Interprete.Interprete: Interficie de usuario que nos permite manejar las sondas, los ejecutores, los dispensadores, las salidas y la cola de ejeucion.
    • SC12KModulo.SC12KModulo: Plantilla basica para los modelos de los modulos dispensadores, ejecutores y escritores..
      • DispensadorIPv6.DispensadorIPv6: Modelo para modulos que proporcionen direcciones IPv6.
        • D_IPv4MappedIPv6.D_IPv4MappedIPv6: Implementacion de un DispensadorIPv6 que genera direcciones IPv4-Mapped IPv6.
        • D_IPv4TEST.D_IPv4TEST: Implementacion de un DispensadorIPv6 que genera direcciones IPv4 para realizar tests en local.
      • EjecutorSondeo.EjecutorSondeo: Modelo para modulos que implementen algun tipo de rastreo sobre 1 direccion IPv6.
        • EjecutorSYN.EjecutorSYN: Implementacion de un rastreo de tipo SYN.
        • Ping.Ping
    • Sonda.Sonda: Union de un dispensador IPv6 y un Ejecutor que permite hacer restreos IPv6 y generar unos resultados.
    • threading._Verbose
      • threading.Thread: A class that represents a thread of control.
        • Trabajador.Trabajador: Thread personalizado con senales de control para el rearranque y el control de la cola de ejecucion.
